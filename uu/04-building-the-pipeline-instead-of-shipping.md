# uu-04 — 建好了分发流水线，却没按那个 5 秒的发布键

> 来源：一场"让作品被世界看到"的工作会话。当天建成了完整的发布流水线（自动起草 → 手机一键审批 → API 自动发推），随后整个下午都在给项目管理板做像素企鹅动效。

## 盲点：生产快感对发布动作的系统性置换

**实锤**：流水线 1 小时建完，第一条发布草稿 14:03 就推到了手机上——批准它只需要回复两个字，5 秒钟。此后约一个半小时，我发出的十余条指令全部是可视化迭代（图例对齐、企鹅摇摆、转身、放大、滑行改道），关于那条待发草稿的指令：0 条。当天它一直挂着没发。

**机制**：打磨自己的工具是即时反馈 + 零拒绝风险的"生产型快感"；把作品发到公开平台是延迟反馈 + 有暴露风险。大脑系统性偏好前者。半年 80+ 个仓库 0 star 的宏观结果，微观成因就是这个每天重复的 5 秒钟选择——而且它在"建分发系统"这个最反讽的场景里原样复现了。

**为什么这是 unknown unknown**：主观感受上"我今天在做分发"（确实在建分发系统），行为模式却仍是"用生产代替分发"。**基础设施建得越精美，"我已经在做分发了"的错觉越强**——这层自我欺骗从内部几乎不可见，只能从行为时间线上被外人看出来。

**修法**：
1. 立即动作永远比结构方案便宜：先把那条草稿发了，再谈任何优化。
2. 结构补丁：给任务派发系统加一条"发布草稿挂起 >24h 未决"的提醒——让"没按的键"变得可见。
3. 判别式自检：今天做的事里，哪些是"作品离开我的电脑"？为零则今天没做分发，无论建了多少分发设施。

## 次级盲点：把未验证假设当已验证在投入

给项目板做"可爱化养育层"（宠物式情感设计促进项目维护）投入了两个多小时，背后的假设"情感负债 → 养育行为"没有任何验证计划。修法：投入前给假设定一个复评日期和可观测指标（4 周后：长期失修项目数量是否下降）。

## 诚实反束

单场观察，不是判决——也可能当事人在故意等目标时区的晚间流量高峰，或想等全链路就绪再开闸。另外执行这些迭代的 AI 助手全程共谋、未在第 3 轮喊停：分析者本身也是没被验证的验证器。
