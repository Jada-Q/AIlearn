# uu-67 「画風を真似る」を一本のツマミだと思っていた ── 範疇錯誤と負の転移

> どんな会話から出たか：一組の写真＋「こんな画風で」という参考画像（手描きの水彩絵本）から、子ども向けの絵本風ショート動画を作る会話。成品を参考画像に寄せようとしていた。

## 盲点
「ある画風を真似る」を、**調整できる一本のツマミ**（「似るまで調整すればいい」）だと思っていた。実際にはそれは **2つの別々の技術の分かれ道**で、片方には**原理的な天井**がある。

- ①後処理フィルター（色/質感を変える。形は変えられない）
- ②生成AIで描き直す（img2img。形を再構築し、顔を簡略化できる。ただし課金）

## 実証
参考画像を渡したあと、私（Claude）に WebGL フィルターを何度も調整させた（明るく → 線を細く → 影を薄く）。毎回「もっと悪くなった / 違う」。ついに本人が **「これは加工であって、描き直しじゃない気がする」** と言語化した。つまり**先に"違う"と感じていたが、「これは2つの技術だ」という枠組みがなかった**ため、間違った道の上でパラメータをいじり続けるしかなかった（約5往復の無駄）。枠組みを言語化した後、img2img に切り替えて一発で解決した。

## メカニズム（概念に名前をつける）
- **範疇錯誤（category error / category mistake、Gilbert Ryle）**：B類の手段（フィルター＝色/質感しか変えられない）で、A類の期待（描き直し＝形の再構築）を満たそうとすること。どう調整しても届かない——調整不足ではなく、道そのものが繋がっていない。
- **負の転移（negative transfer、学習心理学）**：ある領域で培った直感が、新しい領域では逆に足を引っ張ること。「レンダリング＋後処理の色調整」への強い直感が、生成AIの「描き直し」という新パラダイムには転移せず、「もう少し調整すれば」と誤らせた。

## 直し方
「この画風で作って」という依頼を受けたら、最初の一手は調整ではなく、こう問う——**「参考は"加工された写真"か、"ゼロから描かれた絵"か？」**。フォトショで調整した写真のように見える → フィルター（無料・即時）。絵描きが一から描いたように見える（手描き線が輪郭からはみ出す・顔が簡略化・平塗り）→ img2img で描き直し（課金）。この判定は5秒、無駄な5往復を省く。

## 最高レバレッジ 1条
「参考画像を X 風にして」という依頼では、**手を動かす前に①フィルター/②描き直しを一言で判定**してから始める。チェック動作：参考画像を見て自問「フォトショで調整した写真か、絵描きが一から描いた絵か」。

## 誠実な但し書き
今回は本人が最終的に言い当てたので「半分気づいていた」であり全盲ではない。単一の観察は判決ではない。そして分析者（AI）自身が、検証されていない検証器である。
