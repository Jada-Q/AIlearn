# uu-24 — 当「学习」变成「生产学习材料」：把 ship 偷换成 build 的盲点

> 这条来自一场「配置开发工具 + 学终端/JSON 基础」的真实会话。主角是本系列作者自己。

## 核心盲点：用「生产/归档动作」冒充「真学习」

**现场实锤**：这场会话里，她做了一大堆「看起来在学习」的动作——听讲解、存 2 条复习笔记、发 1 篇科普长文、最后还触发归档命令。但全场**两道用来验证理解的预测题，一道都没真答**：第一道开放题（「A 会发哪个项目、B 会发哪个」）被当成选择题回了个「A」；第二道直接跳过，转头去做归档。

**机制**：整理、输出、归档都有**即时成就感**，而真正的验证（自己复述一遍、预测一个结果）既枯燥、又有「当场答错」的风险。大脑于是悄悄拿前者替换后者。结果是：「学 JSON」的可见产出是一篇科普文 + 一条笔记，而「我能独立读懂一段 JSON」这件事，**从头到尾没被检验过一次**。

**为什么危险**：产出物会伪装成学习成果。一个月后翻到那篇科普文，会以为「这个我学过」——其实只生产过、没掌握过。

**修法**：把「被验证」当成学习的 **ship gate**。学完立刻用最低门槛的方式自检（答一道题 / 复述一句话），**「产出了材料」不算数**。答错不可怕，回避验证才可怕。

## 这不是孤例：同根第 4 次

它跟本系列三条旧账同一个根——**用「探索 / 建造 / 采购」的爽感，替换「交付 / 验证」的踏实**：

- uu-20：build/explore > ship/validate
- uu-22：把忙碌伪装成进展
- uu-23：没有目标地囤工具

这次只是换到了「学习」场景。同一个根出现第 4 次，已经不能用「偶然」解释。

## 次要盲点：不知道自己还能要什么

整场是「AI 提议 → 她 yes/no 裁决」。在自己没有地图的领域（开发工具、基础技术），人只能被动评判别人给的选项，**无法主动设计自己的需求**——于是连「我还能要什么」都不知道。修法：每进一个陌生领域，先让对方给一张「你还能做 A/B/C」的能力地图，把看不见的选择空间摊开。

## 诚实反束

这是单场观察叠加历史模式的推断，**不是判决**。也许她只是赶时间、或那两题一看就懂懒得打字。把单次行为固化成「对一个人的定性」，本身就是本系列警惕的第一个盲点。写下它的人，也是个没被验证的验证器。

## 同场更新（当场反转）

写完这条之后，作者回到被跳过的两道预测题，**连续答对了 JSON 嵌套取值 + 正则边界两题，还把刚学的「边界」原理从原例迁移到一个全新例子**。也就是说：这条盲点在被点破的**同一场**，就被她用行为反驳了——她没让归档翻篇，而是回去接受了验证。

这不推翻盲点作为长期模式（uu-19/20/22/23 的历史还在），但这次反转值得记：**盲点被点破后能当场纠正，本身就是「指出 → 立即验证」这个修法有效的实证**。对时记和错时记要一起留——只留「又犯错」会让档案攒出一个比真实更悲观的人。
