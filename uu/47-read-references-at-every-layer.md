# uu-47 — 参考只读到"长什么样"，漏掉它同时在演示的"怎么做 / 能做到什么"

> 一场用 AI 视频工具做黏土风科普片（「天空为什么是蓝的」）的会话里挖出来的。用户贴了一张官方工具 demo 说"做成这种风格"，结果整场走了手动分镜 + 视频拼接 + 外接配音的弯路——那张 demo 其实同时在演示"一句提示词就能出整片"，没人读到。

## UU：面对一个信息密集的样本，注意力锚在最显眼的维度（视觉风格），其余同等重要、并列存在的维度（工作流 / 能力边界 / 成本）被无声跳过

- **实锤**：贴 demo 时的配文是「做成这种风格解说的」——只提取了**视觉风格**。但同一张图上方明写着「一句短提示词，这个模型就能做出有说服力的解说动画」，下方还附了完整提示词模板。这个"一句话出整片"的**工作流 / 能力**维度没被提取，于是走了一整场手动分镜 → 逐段生成 → 本地拼接 → 外接 TTS 配音的弯路。直到会话最后一条才反应过来"这个平台本身就能生成语音"。样本明明同时在说好几件事，只读到了最亮的那件。

- **机制 · 概念点名**：**选择性注意（selective attention）**——注意力被最 salient 的特征捕获，其余并存的信息未被编码进决策。你盯着"它长什么样"，"它怎么做出来的 / 它能做到什么"就在旁边，却没进脑子。

- **修法**：拿到任何"参考 / demo"，回车前先写一行三栏——**风格＝__ / 做法＝__ / 能力＝__**。三栏填不满＝只读了一层，回去再看一遍原样本。本场若开局就填，"做法＝一句提示词出整片"当场会跳出来，整条弯路可省。

## 最高杠杆 + 立即可做

把三栏当**看参考的标准动作**，任何"参考某样本做 X"之前先套一遍：

> **风格**（视觉 / 语气）＝__ ｜ **做法**（它是怎么被生产出来的）＝__ ｜ **能力**（这条路能到什么边界 / 成本）＝__

对照本场反例：只填了"风格＝橡皮泥定格"，"做法""能力"两栏空着——空着的那两栏，正好就是后来翻车、绕路的地方。**卡在哪、绕在哪，往往就是当初没填的那一栏。**

## 诚实反束

这场弯路的**主因是 AI 助手（本分析的作者）没实测就断言"平台原生配音不适合旁白"**，把用户导上了手动路。所以把"没提取工作流"记成用户的盲点，有甩锅嫌疑——更公允的说法是**共同盲点**，且做这份分析的验证器本人，正是制造弯路的那一方。单场一次观察不是判决；分析者是没被验证的验证器，这条判断请打折收下。
