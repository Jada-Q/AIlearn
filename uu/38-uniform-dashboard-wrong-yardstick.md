# uu-38 · 统一的仪表盘，诱导你用同一把尺子量所有东西

> 来源：一场"给两个客户网站装 Google 搜索后台（Search Console）、并第一次学着看这些数据"的多日会话。我对着一个满屏是 0 的仪表盘反复焦虑，才被看出这个洞。

## 认知盲点：用"转化型"指标去量"门面型"的东西

我给一个设计事务所的网站装好了 Google Search Console，然后对着它连续几天的"0 点击、0 曝光"反复焦虑——"怎么没数据"、"效果在哪看"。

但那是个**门面型网站**（credibility site）：设计事务所的客户是靠口碑、作品集链接、熟人介绍找上门的，几乎不会有人在 Google 搜"建筑设计"然后搜到它。它结构上就不该有搜索流量。

我甚至**早就有一条笔记**写着"建筑/设计/咨询是门面型站，不是转化型站"。但那条笔记停在"这是什么类型的站",从没走到下一步——"所以我该看哪个数字"。知道了分类，没用上分类。

## UU：仪表盘对所有东西显示同一套指标，于是诱导你用同一把尺子

**「效果」根本不是一个统一的东西。** 一个卖货的站，效果 = 成交；一个门面站，效果 = 有人搜你名字时能干净地找到你；一个内容站，效果 = 被引用、被读。三种"效果"是三个不同的数字。

但仪表盘不管你是哪种站，**都给你摆同一套指标**（点击、曝光、排名）。这就诱导你用同一把尺子去量所有站——然后把"这类站本来就不靠这个指标"误读成"没效果 / 失败"。

**机制**：可测量的会挤掉重要的。点击、曝光有一个明确的数字；"可信度""被 AI 引用"没有数字。于是注意力自动锚到有数字的那个。这就是"路灯下找钥匙"——不是钥匙掉在路灯下，是只有路灯下看得见。

**放大版（最扎心的一层）**：我做这套优化，有一部分目标是让 AI（ChatGPT、Perplexity 这类）在回答问题时引用我的网站。而"被 AI 引用"这件事，Google 的仪表盘**完全看不到**。也就是说，我可能一边在为 AI 可见性做优化，一边死盯着一个永远反映不出这份努力的 Google 后台，几周后看到低数字，得出"白做了"的结论——而真相只是：我在看错的仪表盘。

## 最高杠杆的一个动作

给每一个网站，在它的项目笔记里写一行：**"这个站的成功指标是什么？"**

- 门面站 = 有人搜名字时能干净搜到 + 信息正确
- 卖货/接单站 = 询盘量 / 搜索流量（这个才真该盯后台）
- 内容站 = 被引用 + 文章曝光

然后立一条纪律：**打开数据后台之前，先问"这个站该拿哪个数字判断"，而不是默认去看最显眼的那个数字。**

## 诚实反束

这是基于单场观察的假设，不是判决。有个替代解释：我对 0 数据的焦虑，可能只是"第一次用这个工具不熟"，几周后自然就学会分辨了，跟"用错尺子"无关。但"仪表盘用统一指标诱导统一判断"这个结构性诱因是真实存在的，它跟你熟不熟工具无关——再熟练的人也会被最显眼的那个数字牵着走。另外，做这份分析的 AI 本身，也是一个没被验证的验证器。
