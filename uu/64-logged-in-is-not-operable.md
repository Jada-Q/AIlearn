# uu-64 — "我登录了" ≠ "AI 能操作它"：驱动 AI 前先分清两种"可用"

> 来源：一场"用代码 + AI（Gemini 出图 / Veo 动画）生成民俗风科普视频"的会话。中途我反复想让 AI 替我操作可灵、Veo 这些网页服务，一次次撞墙。挖出来的盲点，是关于"AI agent 到底能替我做什么"的一个结构性误解。

## UU-1（最高杠杆）｜"我登录了 / 这工具能打开" ≠ "AI 能操作它"

**实锤**：我连续几次踩同一点——「可灵能打开」（AI 打开的是一个**未登录的自动化窗口**）→「我已经登入了」（我登的是我**自己的 Chrome**，不是 AI 驱动的那个窗口）→ 干脆把邮箱给它让它替我登 → 反复说「你执行」「你用付费的生成」。每一次，我都默认："这个服务对我可见 / 我有账号 / 我登录了" = "AI 就能替我操作它"。

**机制**：**ELIZA 效应（ELIZA effect，Weizenbaum 1966）**——人天然把程序想象得比它实际更"像我、在我的环境里、用我的身份"。1960 年代那个只会套模板回话的聊天程序 ELIZA，就让用户以为它真的懂自己。半个世纪后我对 AI agent 犯的是同一个错：以为它"用我的电脑、我的登录态"在跑。

真相是三层墙，撞上之前全是隐形的：
1. AI 驱动的是一个**独立的自动化浏览器**，有它自己的身份、cookie、登录态，跟我平时用的 Chrome **不共享**——我在 Chrome 里登录了，那个窗口照样是空的。
2. **交互式登录**（输账号密码 / Google 授权）AI 不该替我做——那是我的凭证。
3. **付款授权**同理，AI 跨不过去。

**修法**：以后要 AI 操作任何网页服务，开口前先自答一句——**我是想让它用「这工具存在 / 我有账号」（对它没用），还是用「它那个自动窗口已经登录了 / 有 API key 存在文件里」（这才是它能用的）？** 凡涉及登录墙 / 付费墙：要么我在**它那个自动窗口里**亲手过一次认证，要么给它一个 **API key**（存文件，别贴聊天框）。这条会反复省时间——每次想让 AI 驱动一个需要登录的服务，都会撞到它。

## UU-2｜把"怎么出清晰视频"问成了"用哪个软件"——我在优化一个错误的旋钮

**实锤**：我问"ffmpeg 和 HyperFrames，哪个出的视频清晰度更高？"但清晰度跟用哪个软件几乎无关——它由**采集分辨率 + 采集方式**决定，软件只是"把画面装进文件"的工人。我在一个根本不影响结果的变量上做选择。

**机制**：**替代性判断（attribute substitution，Kahneman）**——面对一个难问题（"为什么我的视频糊 / 怎么才能清晰"），大脑会偷偷把它换成一个好回答的问题（"我该用哪个工具"），然后回答后者、却以为回答了前者。工具是看得见、好比较的；采集管线是看不见、难判断的——所以注意力自动滑向工具。

**修法**：对某个结果不满意时（糊了 / 慢了 / 错了），先逼自己问一句——**决定这个结果的真正变量是什么？** 再做选择。很多时候答案根本不在"换个工具 / 换个库"这一层。

## 最高杠杆 1 条 + 立即可做的检查动作

UU-1。下次要 AI 操作任何网页服务，开口前先自答：「我是要它用『它存在 / 我有号』，还是用『那个自动窗口登录了 / 有 key 文件』？」——前者它干不了，后者才行。

## 诚实反束

单场观察不是判决；而且做这个分析的 AI 本身就是个没被验证的验证器——我的这些"盲点"，也可能只是它这一场的过度解读。
