# 共享环境观察者效应：当你的点击变成 AI 的"灵异 bug"

> 2026-06-12 · unknown unknowns 系列第 1 篇 · 源于一天建成 3 件 WebGL 作品的实战会话

和 AI agent 协作一整天做创意编程（复刻 igloo.inc 粒子效果 → 生成式音乐钢琴 → 线条叙事作品），收尾时让它做"我不知道的不知道"分析。下面是它挖出来的三条——按实锤硬度排序。

---

## 1. 你和 agent 共享工作环境时，你的动作是它的数据污染源

**实锤（同一会话内 4 次）**：agent 用 Playwright 弹出的浏览器窗口做视觉验证，我在旁边把这个窗口当"demo 预览"玩——点了形状切换按钮、调了窗口大小、中途关掉浏览器。结果：

- agent 把我点出来的形状切换当"灵异 bug"追查了好几轮
- 截图截到我翻转的镜头状态，误诊成代码问题
- 录屏脚本两次中断

**机制**：我以为我在"看 demo"，实际我在"扰动正在进行的实验"。agent 的视觉验证默认环境受控，而这个假设双方都没有显式声明过。

**修法（一个动作）**：观赏窗口与实验窗口分离。看效果用自己的浏览器开 localhost 链接，agent 弹出的窗口视为它的无尘室，不碰。

**放大趋势**：多 session / 多 agent 并行时，"谁的动作进了谁的观测"会指数复杂化——今天是截图误诊，明天可能是两个 agent 互相把对方的操作当 bug。

## 2. 语音输入依赖一层你看不见的静默纠错

**实锤**：一天内的语音 typo——"钢铁为主题"（钢琴）、"笔加索示"（毕加索式）、"粗心者"（初学者）、"把数字改成 X"（文字）。agent 全部猜对了，所以这层纠错对我完全不可见。

**风险形态**：猜错的那天，我看到的表象是"AI 做了个莫名其妙的东西"，而真实根因是 typo + 纠错失手——我没有任何机制区分这两者。

**修法**：AI 产出奇怪东西时，第一反应不是"它笨了"，是回看自己上一条消息的原文——"我刚才那句语音转出来到底是什么字"。

## 3. 公开复刻的署名规范是声誉资产，不是礼貌

**实锤**：把 igloo.inc（Awwwards 年度网站）的粒子效果复刻研究 push 到 public repo 时，README 里的 "educational homage + 原工作室致谢 + case study 链接" 是 agent 主动加的，我没有要求——说明这条规范本来不在我的雷达上。

**边界**：技术手法可以复刻，美术、形状、品牌是别人的 IP；复刻件可以进 portfolio 演示"技术能力"，不能当"自己的设计成果"展示。
