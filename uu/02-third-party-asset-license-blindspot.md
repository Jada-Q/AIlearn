# uu-02 — 第三方资产引入没有 license 检查这道门

> 来源：一场 vibe coding 会话。为了消除网页合成器的"电子感"，把一个真乐器采样库（FluidR3 GM soundfont，CDN 直连）接进生产并部署——人和 AI 双方全程零人提 license。事后做盲点复盘才查：CC-BY 3.0，要求署名，部署时已处于违反状态（发现后补署名修复）。

## 盲点本体

**第三方资产（音频采样 / 字体 / 图标 / 3D 模型 / 数据集）进项目时，"license 允许我这种用途吗"不在任何人的默认检查清单里——连"它是个需要检查的问题"都没意识到。**

**实锤**：接入采样库之前，技术验证做得非常足——curl 确认每个 URL 返回 200、校验 mp3 magic bytes、用不存在的文件做阴性测试确认 403。但这些全是"技术上能不能用"，没有一秒花在"法律上能不能用"。直到归档流程顺手翻了一眼源 repo 的 README 才看到 CC-BY 条款，而项目已经带着未署名的采样上了生产，public repo 里还写着 "MIT."

**机制**：为什么这个盲点能稳定存在——

1. license 违规是**唯一一种完全静默的缺陷**：不报错、不 404、不出现在 console、不影响任何功能。所有靠"出错信号"驱动的检查习惯对它全部失效
2. vibe coding 的流速奖励"接上能响就继续"，而 license 检查没有任何自然触发点
3. 后果不对称：大多数时候没人追究，但 CC-BY-SA / GPL 这类传染性条款一旦混进商用项目，污染的是整个 repo 的 license 状态

**修法**：不新建习惯，把它**挂在已有的验证动作上**——curl 验证第三方资产可用性的那一刻，顺手多查一条 license（通常就在同一个 README 里，30 秒）。三档速判：

- CC0 / MIT / Apache → 随便用
- CC-BY → 用了就署名（README 一行）
- CC-BY-SA / GPL / 没标 license → 停下来想，默认不进商用项目

## 诚实反束

基于单场对话的假设，非判决：license 意识可能存在只是没说出口；"挂在验证习惯上"的修法是否真能触发，要看下次引入资产时的行为——分析者本身也是没被验证的验证器（这次 AI 自己同样漏了，直到归档流程才补查）。
