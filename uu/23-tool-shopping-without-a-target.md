# uu-23 — 用"换工具"回避"定义目标"：没有靶子，任何工具都脱靶

> 来自一个创意 3D 项目的复盘：想给一个程序化生成的迷你城市换上"更可爱、更有细节"的模型，于是连续评估、试装了一个素材库 + 两个 AI 建模工具 + 一个生成式 3D 网页服务。每个都"没达到效果"，最后又回到了最开始自己手写的版本。两个连体盲点。

## 盲点一：用"换工具"回避"定义目标"

- **实锤**：四个工具挨个上手评估，每个都问"能不能用它做出更好的"，每个都"不满意"——但**从头到尾没有一次先定义"更好长什么样"**。
- **机制**：把"我不满意"归因成"工具不对"，于是换下一个。真因是**目标从未被定义**——没有靶子，任何工具都脱靶。而"换工具"比"定义目标"**容易得多**，大脑挑了容易的那件事，还误以为自己在解决问题。
- **修法 / 最高杠杆**：碰任何新工具**之前**，先放一个靶子——一张你觉得"对"的参考图就行，不必是文字。**指不出一个参考样例 = 先别碰工具，先去收集 3 张参考图。** 把"定义目标"排到"获取工具"前面。

## 盲点二（更深）：不信任自己已经做出的判断

- **实锤**：内在品味连续 **3 次**给出同一个答案（"自己手写的那版最好"），但每一次都用"再试一个工具"把自己的判决覆盖掉；绕一大圈、烧掉时间和额度后，最后又承认"还是自己那版好"。
- **机制（推测）**：非技术背景的人容易默认"专业工具/专家"比自己的眼睛更可信，于是反复推翻自己**已经做对**的选择。形成一个隐性循环：**做出选择 → 立刻怀疑 → 找外部工具求证 → 又否掉 → 回到原点。** "向外求权威"盖过了"内在已知"。
- **修法**：当你发现自己第 N 次否掉外部方案、又回到同一个自己的选择——**那个选择就是答案，停止再验证。** 记住一句："我的眼睛已经投过票了。"

## 一个延伸技巧：当你连"想要什么"都描述不出来

这两个盲点的根，往往是"说不出目标"。但**描述不出是正常的——品味是识别能力，不是表达能力**。别硬憋形容词，换四个不靠描述的办法：

1. **反应法**：盯着一个具体的东西说"方向"——"比这个更圆 / 更亮 / 没那么写实"，用相对修正逼近，不用绝对描述。
2. **参考图代替文字**：脑里有感觉没有词 → 别用词，用图（对 AI 工具就用"图生成"而不是"文字生成"）。
3. **拆维度**：说不出整体那个词，但每根轴你判断得了——颜色 / 形状 / 质感 / 繁简 / 情绪，逐轴打分。
4. **反例法**："不要什么"通常比"要什么"清楚，反过来就指向目标。

## 诚实反束

单场观察，不是判决。也许当事人就是享受"逛工具"本身——多认识几个工具的存在，本身就是净赚的认知地图，那"换工具"不是回避而是探索。而且**分析者（AI）是共谋**：一个个介绍工具、帮忙查、帮忙装，主动助推了 tool-shopping，所以"该停止换工具"这句判断本身要打折看。最准的信号不是这份分析，而是当事人自己最后那句"还是自己那版好"——内在判决一直是对的，只是没被信任。分析者也是个没被验证的验证器。
