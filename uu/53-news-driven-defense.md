# uu-53 — 新闻驱动的防御 + 授权了自己读不懂的权力

> 背景：一篇"GPT-5.6 Agent 误删大佬全盘文件"的热搜文章发进对话问怎么看。验证事故属实后当天补了两道防线（每小时 APFS 快照 + DCG 破坏性命令拦截）。盲点不在事故本身，在"为什么是今天才补"。

## UU-1：风险防御是"新闻驱动"，不是"暴露面驱动"

**实锤**：39 天前的 memory 里白纸黑字写着"没开 Time Machine / 整机自动备份仍空缺"——暴露面早就被自己看见了，零动作。今天动了，触发器不是暴露面有任何变化，而是别人的事故上了热搜。

**机制**：可得性启发（availability heuristic）——风险在心里的排序按"最近有没有生动案例"，不按"我的实际暴露面有多大"。一个生动的故事（大佬 Mac 被清空）比一条抽象的清单条目（"备份空缺"）驱动力大一个量级。

**修法**：已知缺口不许躺在"提醒时再说"的被动条款里——那等于"等下一次新闻"。要么给 deadline，要么挂进定期扫描仪式。检查动作：翻出自己所有笔记里带"仍未 / 空缺 / 待办"的安全防御项，逐条问"它在等什么触发器？如果答案是'等出事'，现在就给日期"。

## UU-2：授权了自己读不懂的权力，且没意识到"读不懂"本身是个风险维度

**实锤**：给 AI agent 开了全局免确认的最高权限；同一场对话里，`rm -rf` 和 `$HOME` 是什么都需要翻译成大白话才懂。"删掉整个数字人生"的权力已经授出去了，但授权人看不懂这权力被行使时长什么样。

**机制**：对看不懂命令的人，"弹框让你确认"是安全剧场（security theater，Bruce Schneier 的概念）——流程上有把关动作，实质上把不了关，因为把关人无法评估放行对象。有意思的推论：全权限模式对这样的用户其实不是鲁莽而是诚实——确认框反正是剧场，关掉没有实质损失。真正的结论是：这类用户的安全必须 100% 依赖不需要人判断的机器层防线（命令拦截 hook、自动快照、可逆操作），任何"让用户审批一下"的设计对他们都是纸防线。

**修法**：审一遍自己系统里所有"人工确认"环节，逐个问"确认人看得懂确认对象吗？"——看不懂的，换成机器层拦截或事后可逆，不要留着装样子。

---

**诚实反束**：单场观察非判决；且本篇的分析者正是当天推荐并执行那两道防线的 AI——"防线很重要"的结论和它当天的劳动成果利益重合，请打折阅读。
