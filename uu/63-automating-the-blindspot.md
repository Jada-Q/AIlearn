# uu-63 — 当你把一个盲点自动化：从「手动只造不发」升级到「造了一台自动挖兔子洞的机器」

> 来源：一场"读完一篇讲 AI Agent 500 天的播客文章 → 让 AI 按'有没有付费信号'把我所有自建工具 triage 一遍"的会话。挖出来的盲点，是我一个早就写进档案的老毛病的**自动化升级版**。

## UU-1（最高杠杆）｜盲点被自动化了 = 盲点升级，不是消失

**实锤**：我早知道自己有"build > ship"（爱造、不爱发/不验证）的毛病，档案里已经记过两次手动版。但这次把自建工具摊开一 triage，三个被标红的项目——一套"自动造 SaaS"的流水线——揭示了新一层：我不再手动挖兔子洞了，我**造了一台自动挖兔子洞的机器**。工厂天天在转，出货口（有人愿意付费的东西）是空的。外部实证还在旁边打脸：一位同行 6000 star 的开源 Agent 都收不到钱，我这台自动出货的工厂就更别指望变现。

**机制**：目标置换（goal displacement，社会学家 Robert Merton）——手段悄悄变成了目的。自动构建器本是"产出可变现产品"的**手段**，但"每天自动产出"这个动作本身，把"产出有人付费的东西"这个**目标**顶掉了。更隐蔽的一层：*"自动化 = 杠杆"* 这个直觉给它盖了一层正当性外衣。真相是——**自动化一件从没验证过 ROI 的事，不是省时间，是更快地制造债务**。

**修法**：给任何"我要自动化 X"的念头加一道前置问——**X 我手动做的时候，有验证过的回报吗？** 没有 → 停。自动化只会把一个没价值的循环放大。立即可做的检查动作：数一下这台机器过去的产出里，我自己回头用过 ≥1 次的有几个；若为 0，ROI 就被实证了，该走关停仪式。

## UU-2｜我需要一面外部镜子才看得见自己的"造 vs 发"模式——从内部看不见

**实锤**：我**已经知道**这个盲点存在（都写进档案了），却仍然需要一篇外部文章 + 让 AI 帮我把工具列表 triage 一遍，才认出它早已物理化成一台机器。"知道我有这个盲点" ≠ "能在它正发生的当下认出它"。

**机制**：内省错觉（introspection illusion，Nisbett & Wilson / Pronin）——人以为自己能直接看到自己的动机和行为，实际上是像旁观者一样**事后推断**的。我以为那台工厂是杠杆，只有外部框架才把它翻译成"盲点的自动化版本"。

**修法**：这正好落在"用强制触发器代替规则"上——"知道但临场认不出"要靠外部结构，不靠自省。给"我要造个工具 / 搭个自动化"设一道 hook 或仪式（比如强制先跑一遍需求验证流程），把外部视角变成结构，而不是指望当场想起来。

## 诚实反束

单场观察不是判决；而且做这个分析的 AI 本身就是个没被验证的验证器——这恰好又是那条元盲点「验证器自身也没被验证过」的一次现身。
