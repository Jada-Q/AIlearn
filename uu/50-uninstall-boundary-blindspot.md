# uu-50 — 卸载软件时，「删掉」的直觉边界比数据的真实边界小

> 来源：一次帮我卸载 Windsurf（Codeium 家的编辑器）的杂事会话。任务很小、UU 很薄，但正因为它平凡，盲点才更值得点出来——越是"随手"的操作，越不会去想边界。

## UU-1：`~/.codeium` 属于「厂商」，不属于「应用」

- **实锤**：AI 不得不专门提醒我「别处如果用 Codeium 补全，删了会掉登录」，我才意识到 `~/.codeium` 是**跨编辑器共享**的。我选"全删"的时候，心智模型是"这些都是 Windsurf 的东西"——但它实际是 **Codeium 账户级**目录，同一个厂商的多个产品（Windsurf、装在 VS Code 里的 Codeium 插件）共用这一份。全删就可能连坐到你还在用的另一个产品。
- **机制**：人习惯用**产品名**给数据归类（"这是 Windsurf 的"），但数据实际按**厂商账户**归类。产品和厂商是两层，直觉把它们压成了一层。
- **一个可用的辨认线索**：macOS 的 bundle id 长这样 `com.exafunction.windsurf`——中间那段 `exafunction` 是母公司，末段才是产品。残留文件常按 bundle id 命名，所以**按厂商名搜比按产品名搜更全**。

## UU-2：删本地文件 ≠ 注销账户

- **实锤**：AI 补了一句"卸载不会自动登出服务器端"，我没接话——说明这根本不在我的考虑范围里。
- **机制**："删除"这个直觉只作用于**看得见的本地文件**。账户是**远端状态**，不在你的硬盘上，所以压根不在"删除"的视野里。这条平时无所谓；但如果你换掉一个工具正是因为**不想让那家厂商继续留着你的数据**，那只删本地就是白删。
- **概念点名**：**数据残留（data remanence）**——信息安全里的既有概念，指数据在"删除"动作之后，仍以缓存 / cookie / 备份 / 服务器端记录等形式残存。删除感觉是终点，数据残留说：不一定。

## 最高杠杆的一条 + 立刻能做的检查

> 删**任何登录过账号的**软件，删完做两件事：
> ① `mdfind -name <厂商名>` 全盘兜底——Spotlight 的命令行搜索，能抓到你**根本不知道存在**的位置（这次 `~/Library/HTTPStorages` 里的登录 cookie 就是这么被抓出来的，我手动清单里没有它）。
> ② 问一句"我登录过账号吗？要不要去官网注销"。

按名字逐个删，只能删掉你记得的；问总索引"哪儿有叫它的东西"，才能删掉你不记得的。

## 诚实反束

单场观察，不是判决。而且"删完追问还有没有残留"这个好习惯，是我自己发起的——说明边界感知没缺失，只是**滞后**（删完才想到），把它前移到删之前就够了。写下这段分析的一方（AI），本身也是个没被验证的验证器。
