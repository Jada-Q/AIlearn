# uu-56 — 首版验收里没有质量维度：功能过了 ≠ 质量看过了

> 背景：一场"复刻 scroll-world 效果"的创意编程会话——Three.js 样条飞行 → 录屏 demo → 双语解说成片。视频交付了四轮之后我才说"有点模糊"，此时返工链已经是：重录母带 + 中英文两版重渲染 + 封面重做。

## UU-A：质量维度不在首版验收里，痛了才浮现

- **实锤**：模糊问题从第一版 demo（720p 实时录屏）就存在。我连续验收通过了 demo、中文成片、英文成片三个交付物，第四轮才提出"视频比较模糊"。更扎心的是，正确解法（"不能从 code 直接出视频吗"——逐帧离线渲染）是我自己提出来的：答案一直在我手里，连接却在痛了之后才发生。
- **机制**：首次验收时注意力全被功能和内容占据（字卡出没出、旁白对不对齐），清晰度、音质、节奏这类质量属性是"看多几遍才浮现"的维度。软件工程里有对应概念：**非功能性需求（non-functional requirements）**——功能需求验收通过，不代表非功能需求被验收过，后者常常到上线后才被发现没人管。
- **修法**：任何视觉/音频交付物的**第一版**到手时，强制 30 秒过四问——**清晰度？声音？节奏？首尾帧？**——不满意当场说。把质量返工从第 N 版压回第 1 版，返工成本差一个数量级。

## UU-B：登录墙被我当成终态，而不是待诊断的故障

- **实锤**：素材站要登录才能下载，我说"下载不了不能登入"——六个字之后零细节：没账号？密码忘了？验证码不过？没有定位，直接等 AI 换路。对比同一场里"视频不推进"的播放问题，我立刻报症状并一路跟到修复。
- **机制**：系统内的问题被我分类为"可修的"，外部服务的摩擦被分类为"不可抗力"。但免费 3D 素材生态（Sketchfab、Meshy）几乎全在登录墙后面——这个分类方式会持续把整个素材生态挡在外面。
- **修法**：登录类阻塞先花 2 分钟说清卡在哪一步（没账号 / 密码 / 2FA / 页面报错）。说得清的多半可修；真不想注册也可以，但那是一个决策，不是一个故障。

**最高杠杆**：UU-A。立即可做的检查动作：下次任何视频/设计类交付第一版到手，先过那四问，再说"可以"。

**诚实反束**：两条都来自单场观察，不是判决；UU-B 也可能只是当时不想折腾账号（合理选择被分析者读成了盲点）；而做这份分析的 AI 本身，是个没被验证过的验证器。
