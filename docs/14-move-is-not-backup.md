# 搬到硬盘 ≠ 备份：一人公司的数据安全系统

> 2026-06-04 · 从一次真实事故（清桌面搬文件丢了 9 个文件）出发，补出的一整套数据安全打法

清理电脑、搬文件到移动硬盘，是每个人都会做的事。但一个看似无害的"搬运"操作，可能真的让你的数据永久消失——而且你以为自己"备份"了，其实没有。这篇从一次实际事故出发，串起一人公司 / 个人创作者该懂的数据安全系统。

---

## 一、最致命的认知误区：「搬移」不是「备份」

| | 搬移（move）| 备份（backup）|
|---|---|---|
| 本质 | 文件挪个地方，**还是一份** | 复印一份，**两份以上** |
| 数据份数 | 1 份 | ≥ 2 份 |
| 一处出事 | 永久没了 | 另一处还在 |

把文件从电脑"搬"到移动硬盘，电脑里就删了——数据**还是只有一份**，只是换到了一个更容易坏、丢、被偷的地方。

**搬移不但不等于备份，反而制造了新的"孤本"**：本来电脑里至少有一份，搬走后唯一一份悬在移动硬盘上，风险反而更高。

## 二、备份的黄金法则：3-2-1

数据备份的行业通用标准，记住三个数字：

- **3** 份副本（原件 + 两份备份）
- **2** 种不同介质（如：本机硬盘 + 移动硬盘 + 云）
- **1** 份异地（off-site，云就算异地——家里进水失火，云上的还在）

任何重要数据，拿这三个数字一套就知道自己缺什么。比如"本机一份 + 云一份"= 2 份 2 地，还差一块物理备份盘才到 3-2-1 满配。

## 三、上云怎么不泄密：自己掌握钥匙

**关键事实**：Google Drive、iCloud、Dropbox 这些主流云盘**都不是端到端加密**。传输和存储确实加密了，但**解密的钥匙在平台手里**——它在技术上**能够**读取你的文件（用于扫描违法内容、合规、响应法律调取）。

这不是哪家特别坏，是大众云盘的行业常态。

**解法**：上传前自己先加密，把"平台也有一把钥匙的保险箱"变成"只有你有钥匙的保险箱"。

macOS 原生做法（不用装任何东西）：

1. 打开「磁盘工具」
2. 菜单：文件 → 新建映像 → 来自文件夹的映像
3. 选要加密的文件夹
4. **加密（Encryption）选 256 位 AES**，设密码
5. 映像格式选「压缩」
6. 生成一个加密的 `.dmg` 文件，上传到云

这样云端那份就是一坨乱码，平台和黑客都看不到内容，只有你的密码能解开。

> ⚠️ 代价：**密码忘了 = 这个加密包永远打不开，谁也救不回**。必须记进密码管理器。强加密没有"找回密码"。

**AES-256** 是全球银行、政府都在用的加密标准，正确使用下实际无法暴力破解。

## 四、设备的全生命周期加密

数据安全不只是文件层面，也是设备层面：

- **在用阶段**：开启 **FileVault**（macOS 全盘加密）。电脑丢了/被偷，别人开不了机、也读不出硬盘数据。
- **销毁 / 转手阶段**：用「系统设置 → 通用 → 转移或重置 → **抹掉所有内容和设置**」，**不是**磁盘工具里的 Erase 按钮。

为什么不用磁盘工具 Erase：它只抹硬盘内容，**不会**退出你的 Apple ID、关闭激活锁（Activation Lock）。结果是你的账号还绑着这台机器，下家可能被锁死。「抹掉所有内容和设置」一步做完：清数据 + 退 Apple ID + 关激活锁 + 回出厂状态。

类比：磁盘工具 Erase 像"把房间东西扔光"，但门锁还是你的钥匙、地址还登记在你名下；「抹掉所有内容和设置」是"扔光东西 + 退租 + 换锁 + 销户"，一步到位。

## 五、反面教材：跨盘搬文件怎么会静默丢数据

事故根因（也是最值得记的避坑点）：

- **exfat 格式的移动盘建不了含 emoji / 特殊字符的文件名**（exfat 是 Windows/Mac 通用格式，移动硬盘出厂常是它）
- 搬运工具（如 rsync）遇到这种名字会**跳过**这些文件
- 如果用了"边搬边删源"的参数（如 `rsync --remove-source-files`），文件没搬过去、源却被删了 → **静默丢失**
- 没有备份的话，这就是**不可逆**的

**正确做法**：跨盘搬运一律走「**复制 → 核对文件数 + 总字节数完全一致 → 才删源**」，任何一项对不上就停下保留源，绝不"边搬边删"一把梭。

## 六、工具箱

| 工具 | 作用 |
|---|---|
| 磁盘工具 / `hdiutil` | 把文件夹加密打包成 AES-256 的 `.dmg` |
| FileVault | 内置盘全盘加密 |
| `yt-dlp` | 视频丢了，按视频 ID 重新下载 |
| 软链接（symlink） | 大文件搬到外接盘、本机只留一个"指针"，省空间又不断引用 |

## 一句话总结

**搬移省的是空间，备份保的是命；上云前自己加密，钥匙攥在自己手里。**

数据安全不是"买块硬盘"就完事，是一整套习惯：**分清搬移和备份 + 守住 3-2-1 + 全程加密**。等到丢了才懂，往往已经不可逆。
