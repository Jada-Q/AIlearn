# 假绿灯：TDD 防「算错」，smoke test 防「环境不对」

> 2026-06-10 · 一次「终端全绿、双击就炸」的复盘

## 事故

一个自己写的 YouTube 下载小工具（AppleScript droplet 打包成 .app，内部调 yt-dlp + ffmpeg）。某天突然下载失败，日志只有一句没信息量的「视频下载失败」。

但磁盘上其实留下了两个文件：一个纯视频流（av1），一个纯音频流（opus）。两个都下全了，**只是没合并成一个 mp4**。

根因：yt-dlp 把视频流和音频流分开下载后，要靠 ffmpeg 合并。脚本里 yt-dlp 写了绝对路径所以找得到，但 **ffmpeg 是靠 PATH 找的**。而这个 .app 是双击 / 拖拽启动的 —— macOS GUI app 启动时用的是**裸 PATH**（`/usr/bin:/bin`，没有 homebrew 的 `/opt/homebrew/bin`）→ 找不到 ffmpeg → 合并失败。

## 关键：这个 bug 一直潜伏，不是「突然坏」，是「突然被触发」

droplet 从第一天起就没把 homebrew 放进 PATH，ffmpeg 从来就找不到。但 ffmpeg **只在需要合并时才用到**。早先下的视频拿到的多半是单文件格式（视频音频已经在一起），不需要合并 → 不碰 ffmpeg → 一切正常。直到某次下载拿到的是需要合并的分离流，潜伏的雷才炸。

> 潜伏 bug 的特征：代码逻辑完全正确，只在某条「平时不走、这次走了」的路径上才暴露。对写的人和用的人**同时是隐形的** —— 不是「看到了没说」，是它在被触发前根本没失败过。

## 为什么 TDD 防不住这个

直觉反应是「下次写这种工具用 TDD 就能防」。但**对这个 bug，TDD 会给你假绿灯**。

TDD 的单元测试跑在**测试环境**里。而你跑测试的环境（终端）恰好 PATH 有 homebrew、ffmpeg 找得到。所以：

- 测试：全绿 ✅
- 真实的 .app：照炸 ❌

测试环境和事故环境不一样，测试就证明不了事故不会发生。这正是「在测试环境通过、在部署环境失败」那一整类 bug —— TDD 的射程打不到。

## 两轴模型：TDD 防「算错」，smoke test 防「环境不对」

| | 防什么 | 例子 |
|---|---|---|
| **TDD / 单元测试** | 逻辑 bug：算错了，且错在数据里肉眼看不出 | 去重算法把 1120 条压成 561 条对不对；srt 时间戳解析 |
| **Smoke test（真实环境跑）** | 环境 / 集成 bug：逻辑对，但运行环境不对 | 双击 app 找不找得到 ffmpeg；权限、PATH、login/wake 事件 |

一句话：**TDD 防「算错」，smoke test 防「环境不对」。** 这次坏在环境，所以该补的是 smoke test，不是 TDD。

## 所以不是「要不要 TDD」，是「用在哪一层」

把这类下载工具拆开：

- **纯逻辑层**（输入 → 算 → 输出，错了肉眼看不出）：去重算法、srt 解析、分块逻辑 → **TDD 合适**
- **编排 + 环境层**（调外部工具、PATH、文件移动、GUI 启动）：下载主流程 → **TDD 假绿灯，要在真实运行环境跑 smoke test**

单靠 TDD 会漏掉这次这种环境 bug；单靠 smoke test 会漏掉去重算坏了那种逻辑 bug。**两者配合才覆盖全。**

## 落地的防护

把一条 **运行时工具可达性检查** 加进了项目的 pre-push hook：复刻 GUI app 的裸 PATH 环境，叠加脚本自己 export 的 PATH，确认 yt-dlp / ffmpeg / claude 都还能解析到绝对路径。push 前就挡住「终端能跑、双击炸」。

写这种检查器有个纪律：**先在阴性样本上验它会失败**。我先在「无 export 的裸 PATH」下跑，确认它正确报「找不到」，再确认修好后的状态能过 —— 否则你可能写了个永远返回 pass 的假检查器，比没有还危险。

而修代码本身的纪律：**GUI app 启动的脚本，所有外部工具都用绝对路径 / 显式 `--xxx-location` 传，不靠 PATH**。而且当你给一个工具钉死路径，要把它**调用的下游工具一起钉** —— 这次的洞就是「钉了 yt-dlp，忘了它内部调的 ffmpeg」。

## 一句话带走

- 写测试前先问：我要防的是「算错」还是「环境不对」？选错工具 = 假绿灯。
- 终端能跑 ≠ 双击 app 能跑。环境 bug 要在**真实运行环境**验。
- 检查器上线前先在阴性样本上确认它真会失败，否则它可能一直在骗你。
